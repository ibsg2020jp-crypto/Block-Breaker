# 要件定義書：ブロック崩し

## 1. プロジェクト概要

### 1.1 プロジェクト名
ブロック崩し

### 1.2 作成するもの
ブラウザで動作する、1人用のブロック崩しゲームWebアプリ。

スマホ、PC、タブレットのブラウザで遊べることを前提とし、ログインや外部サーバーを必要としない静的Webアプリとして実装する。

### 1.3 ゲームジャンル
パズル寄りのステージ制ブロック崩し。

単純にボールを跳ね返してブロックを壊すだけでなく、ステージごとのギミック、短い会話、スコア、実績、タイムアタック要素によって達成感を出す。

### 1.4 重要視すること
- スマホでも操作しやすいこと
- 画面がシンプルで見やすいこと
- 軽量で、ブラウザだけで動くこと
- 難しい操作や専門知識なしで遊べること
- 後からステージやギミックを追加しやすいこと

### 1.5 避けること
- ログイン必須
- 外部サーバー必須
- 重い3D処理
- 複雑すぎる画面
- 操作方法が多すぎる設計
- 専門知識がないと編集できない構造

## 3. ゲーム概要

### 3.1 基本ルール
プレイヤーは画面下部のパドルを左右に動かし、ボールを跳ね返してブロックを壊す。

ボールは壁、パドル、ブロックに当たると反射する。
全ての破壊対象ブロックを壊すとステージクリア。

### 3.2 ゲームの目的
- 各ステージの全ブロックを破壊する
- ステージを進めて物語を進行させる
- 高スコア、短時間クリア、実績解除を目指す

### 3.3 クリア条件
ステージ内の破壊可能なブロックを全て破壊する。

### 3.4 失敗条件
以下のいずれかで失敗とする。

- 制限時間が0になる
- 残機が0の状態でボールが画面下に落下する

### 3.5 1ステージの長さ
1ステージは約1〜3分を目安にする。

序盤は60〜90秒程度でクリア可能な難易度にし、後半ステージでは2〜3分程度まで広げる。

## 4. 操作仕様

### 4.1 スマホ・タブレット操作
- 画面下部をドラッグしてパドルを左右に移動する
- タップでボール発射
- 会話やリザルト画面はタップで次へ進む

### 4.2 PC操作
- マウス移動またはドラッグでパドルを左右に移動する
- クリックでボール発射
- キーボード操作にも対応する
  - `←` / `→`：パドル移動
  - `Space`：ボール発射、決定
  - `P`：一時停止

### 4.3 操作方針
スマホを優先し、細かいボタン操作を必要としない設計にする。

## 7. ゲームシステム仕様

### 7.1 パドル
パドルは画面下部に配置する。

仕様：

- 横移動のみ
- 画面外には出ない
- スマホではドラッグ位置に追従する
- PCではマウスまたはキーボードで移動する

パドルに当たった位置によってボールの反射角を変える。

- 中央に当たる：上方向へまっすぐ反射
- 左端に当たる：左上へ強く反射
- 右端に当たる：右上へ強く反射

### 7.2 ボール
ボールは壁、パドル、ブロック、ギミックに当たると反射する。

仕様：

- 初期状態ではパドル上に固定される
- タップまたはクリックで発射される
- 左右の壁に当たると左右方向に反射する
- 上の壁に当たると上下方向に反射する
- 下に落ちると残機が1減る
- 残機が残っていればパドル上に再配置される
- 残機が0の状態で落下すると失敗

### 7.3 反射処理
基本反射は以下の考え方で実装する。

- 左右の壁に衝突：`vx *= -1`
- 上の壁に衝突：`vy *= -1`
- パドルに衝突：当たった位置から反射角を計算
- ブロックに衝突：衝突方向に応じて `vx` または `vy` を反転

ボールがブロックや壁にめり込まないように、衝突後は位置補正を行う。

### 7.4 ブロック
ブロックはステージごとに配置する。

ブロック種別：

| 種別 | 内容 |
|---|---|
| 通常ブロック | 1回当たると破壊 |
| 硬いブロック | 2〜3回当たると破壊 |
| 壊れないブロック | 反射のみ。クリア条件には含めない |
| スコアブロック | 壊すと高得点 |
| スイッチブロック | 壊すとギミックが作動 |
| ワープブロック | ボールを別地点に移動させる |

### 7.5 スイッチギミック
スイッチブロックを破壊すると、ステージ内の状態が変化する。

例：

- 壊れないブロックが通常ブロックに変化する
- 隠しブロックが出現する
- ワープ地点が有効になる
- 残り時間が増える

### 7.6 ワープギミック
ボールがワープ入口に触れると、対応するワープ出口へ移動する。

仕様：

- ワープ後もボールの速度は維持する
- 連続ワープを防ぐため、短いクールタイムを入れる
- ワープ入口と出口は色や記号で対応が分かるようにする

### 7.7 スコア
スコア加算例：

| 行動 | 点数 |
|---|---:|
| 通常ブロック破壊 | 100 |
| 硬いブロック破壊 | 200 |
| スコアブロック破壊 | 500 |
| ステージクリア | 1000 |
| 残り時間ボーナス | 残り秒数 × 10 |
| 残機ボーナス | 残機数 × 300 |

### 7.8 評価ランク
ステージクリア時に評価を表示する。

評価基準：

| ランク | 条件 |
|---|---|
| S | 目標タイム以内、残機減少なし、高スコア |
| A | 目標タイム以内でクリア |
| B | 通常クリア |
| C | クリアしたが時間または残機に余裕が少ない |

最初の実装では、タイムと残機を中心に評価する。

### 7.9 タイムアタック
各ステージの最速クリアタイムを保存する。

クリア画面とステージ選択画面にベストタイムを表示する。

### 7.10 実績
ブラウザの `localStorage` に実績解除状況を保存する。

実績例：

| 実績名 | 条件 |
|---|---|
| はじめの一歩 | ステージ1をクリア |
| ノーミスクリア | 残機を失わずにステージクリア |
| スピードスター | 目標タイム以内にクリア |
| ブロックハンター | 累計100個のブロックを破壊 |
| ワープ使い | ワープを10回使う |
| 完全制覇 | 全ステージをクリア |

## 8. ステージ仕様

### 8.1 初期ステージ数
最初は5ステージを実装する。

追加しやすい構造にするため、ステージ情報は `src/stages.js` にデータとして定義する。

### 8.2 ステージデータ例

```js
const stages = [
  {
    id: 1,
    name: "はじまりの壁",
    timeLimit: 90,
    lives: 3,
    targetTime: 60,
    dialogueBefore: [
      "ここからブロック崩しの冒険が始まる。",
      "すべてのブロックを壊して道を開こう。"
    ],
    blocks: [
      { x: 1, y: 1, type: "normal" },
      { x: 2, y: 1, type: "normal" },
      { x: 3, y: 1, type: "hard", hp: 2 }
    ],
    gimmicks: []
  }
];
```

### 8.3 初期ステージ案

| ステージ | 名前 | 内容 |
|---:|---|---|
| 1 | はじまりの壁 | 通常ブロック中心の基本ステージ |
| 2 | かたい守り | 硬いブロックを追加 |
| 3 | スイッチ回廊 | スイッチでブロック配置が変化 |
| 4 | ワープゲート | ワープでボールの軌道が変化 |
| 5 | 静かな最終試験 | スイッチとワープを組み合わせた総合ステージ |

## 9. 物語・会話仕様

### 9.1 物語の方針
複雑な長編ストーリーではなく、ステージ進行の動機づけになる短い会話を入れる。

例：

- 不思議なブロックに閉ざされた道を進む
- ステージをクリアすると少しずつ先へ進める
- ギミックを使って障害を突破する

### 9.2 会話の実装方針
会話データはステージデータ内に配列で持たせる。

```js
dialogueBefore: ["セリフ1", "セリフ2"],
dialogueAfter: ["セリフ3", "セリフ4"]
```

会話が不要なステージでは空配列にする。

## 15. 実装時の注意点

### 15.1 衝突判定
ボールが高速化するとブロックをすり抜ける可能性がある。

最初は速度上限を設ける。
必要に応じて、1フレーム内で移動を分割して衝突判定する。

### 15.2 スマホのスクロール抑制
ゲーム画面をドラッグしたときにページ自体がスクロールしないよう、Canvas周辺では `touch-action: none;` を指定する。

### 15.3 画面比率
Canvas内部のゲーム座標は固定基準にし、表示サイズだけを画面に合わせて拡大縮小する。

推奨内部解像度：

```txt
幅: 360
高さ: 640
```

PCではこの比率を維持して中央表示する。

### 15.4 データ追加のしやすさ
ステージ追加時にゲーム本体の処理を変更しなくて済むよう、ステージ定義をデータ化する。

ギミックやブロック種別も文字列の `type` で管理する。

## 16. 完成イメージ

落ち着いた色合いの画面に、シンプルなブロック、パドル、ボールが表示される。

プレイヤーはドラッグだけでパドルを操作し、ボールを反射させてブロックを壊していく。
ステージごとにスイッチやワープが登場し、短い会話によって物語が少しずつ進む。

1ステージは短く、クリア評価やベストタイム更新によって何度も遊びたくなる構成にする。
